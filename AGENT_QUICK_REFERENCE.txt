POSITION SIZING & RISK:
  deployAmountSol          Amount per position (SOL)           [default: 0.5]
  maxPositions             Max simultaneous positions           [default: 3]
  stopLossPct              Stop loss threshold (%)              [default: 5-8]
  maxHoldHours             Force-close position after N hours   [default: 6]
  dailyLossLimitUsd        Daily loss limit before pause ($)    [default: 25]

MARKET ENTRY FILTERS:
  minTvl                   Minimum pool liquidity ($)           [default: 10k]
  maxTvl                   Maximum pool liquidity ($)           [default: 500k]
  maxPoolAgeDays           Reject pools older than N days       [default: 3]
  minVolumeTvlRatio        Minimum volume/TVL ratio             [default: 20]
  minBinStep               Minimum bin step (100 or 125)        [default: 100]
  allowedBinSteps          Allowed bin steps                    [default: [100,125]]

VOLATILITY & MOMENTUM:
  minAtrPctForEntry        Min ATR% for entry signal            [default: 2.0]
  atrPct < 3%              Signal: Low volatility (WAIT)        [threshold]
  priceChangeH1 > 2%       Signal: Bullish momentum (ENTER)     [threshold]
  priceChangeH1 < -5%      Signal: Bearish crash (AVOID)        [threshold]

FEES & YIELD:
  minDailyFeeYieldPct      Minimum daily fee yield (%)          [default: 0.3]
  maxILvsHodlPct           Max IL vs HODL before exit (%)       [default: 15]
  takeProfitFeePct         Fee income take-profit (%)           [default: 5]

EXIT SAFETY:
  volCollapseThresholdPct  Volume collapse exit trigger (%)     [default: 25]
  slCooldownMinutes        Stop-loss cooldown after trigger     [default: 120]
  slCircuitBreakerCount    SL triggers before circuit breaks    [default: 3]
  slCircuitBreakerWindowMin Window for counting SL triggers      [default: 60]
  slCircuitBreakerPauseMin Pause duration after circuit break   [default: 60]

API & DATA STALENESS:
  maxOhlcvStaleMinutes15m  Max staleness for 15m candles        [default: 90]
  maxOhlcvStaleMinutes1h   Max staleness for 1h candles         [default: 180]

WHAT EACH INDICATOR CONTROLS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SUPERTREND (15m + 1h):
  → Controls entry signal detection (BULLISH = ready to deploy)
  → Controls exit decision (flip from BULLISH to BEARISH = close)
  → Adjust period/multiplier in src/utils/ta.js if too noisy/delayed
  → Default: period=10, multiplier=3

VOLUME/TVL RATIO:
  → Controls pool selection quality (minVolumeTvlRatio > 20)
  → Minimum means: must see 20x annual volume through pool to consider
  → Increase if missing good pools; decrease if accepting low-volume trades

ATR (Average True Range):
  → Controls volatility gate (minAtrPctForEntry > 2.0)
  → If ATR too low: IL risk without corresponding fee yield
  → If ATR too high: stop-loss likely to trigger early
  → Safe range: 2–5% for Evil Panda

FEE/TVL RATIO (Daily Fees / Total Value Locked):
  → Controls dead-pool rejection (minFeeActiveTvlRatio > 0.01)
  → If pool making < 1% APY daily fee: not worth the risk
  → Real pools should show 5–20% daily fee APY for DLMM

PRICE CHANGE (1h, 5m):
  → priceChangeH1 > 2% = bullish confirmation (enter)
  → priceChangeH1 < -5% = bearish rejection (avoid)
  → priceChangeM5 > 0 = 15m candle must be green (additional gate)

WHEN TO ADJUST VS LEAVE DEFAULTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

LEAVE DEFAULTS IF:
  ✓ Positions hitting stop-loss < 1 per day
  ✓ Win rate > 40% (fees earned > SL losses)
  ✓ Circuit breaker not tripping
  ✓ No cascading manual review queue

ADJUST IF:
  ✗ Stop-loss hit > 3× per day             → Increase stopLossPct (8→10) or reduce minAtrPctForEntry
  ✗ Missing obvious entry opportunities   → Reduce minVolumeTvlRatio (20→15) or minTvl (10k→5k)
  ✗ Positions staying unprofitable        → Reduce maxHoldHours (6→4) to exit dead positions faster
  ✗ Circuit breaker trips weekly          → Increase slCircuitBreakerCount (3→4) or slCooldownMinutes (120→180)
  ✗ DexScreener API timeouts              → Increase maxOhlcvStaleMinutes15m (90→120)

NEVER ADJUST:
  - allowedBinSteps (locking to 100/125 is intentional — maintains Saklek Mode)
  - maxBinsPerPosition (125 is hardcoded Solana TX limit)
  - activeStrategy (Evil Panda/Deep Fishing chosen by classifyMarketRegime)

EXIT TRIGGER CODES (exit_events.exitTrigger)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

MAIN LOOP (exitTrigger = closeReasonCode):
  STOP_LOSS              Drawdown >= stopLossPct (default 10%) — immediate close
  TAKE_PROFIT            PnL >= takeProfitFeePct — static profit target
  TRAILING_TAKE_PROFIT   Peak–current > trailingDropPct — trailing peak drop
  MAX_HOLD_EXIT          Age >= maxHoldHours (6h) — dead capital cleanup
  OOR_BINS_EXCEEDED      Out-of-range bins >= outOfRangeBinsToClose

WATCHDOG PATHS (exitTrigger ≠ closeReasonCode — ⚠️ split):
  GUARDIAN_ANGEL_DUMP    → DB: GUARDIAN_ANGEL_DUMP_EXIT
  ZOMBIE_EXIT            → DB: ZOMBIE_EXIT_${reason}
  TRAILING_TP_HIT        → DB: TAE_WATCHDOG_EXIT_${zone}
  SUPERTREND_FLIP        → DB: TAE_WATCHDOG_MOMENTUM_EXIT_${zone}
  OOR_BAILOUT            → DB: OOR_HARD_EXIT_WATCHDOG
  PANIC_EXIT_BEARISH_OOR → DB: same (consistent)
  PROFIT_PROTECTION      → DB: PROFIT_PROTECTION_BEARISH

OPERATOR INITIATED:
  MANUAL_CLOSE           /pause + manual close
  AGENT_CLOSE            AI agent decision
  ZAP_OUT                Close + swap all tokens to SOL

CIRCUIT BREAKER (NOT in exit_events — separate table):
  SL_CLUSTER_THRESHOLD_MET → circuit_breaker_events (poolAddress, slCount, pausedUntil)

INCIDENT QUICK CARDS  (use during live volatility)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

QC-1: CIRCUIT BREAKER TRIPPED
  Detect:  runtime-state.json has hunter-circuit-breaker.pausedUntil key
           Telegram "⚡ CIRCUIT BREAKER TRIPPED" | logs slCount >= slCircuitBreakerCount
  NOTE:    SL_CLUSTER_THRESHOLD_MET writes to circuit_breaker_events table (NOT exit_events)
           Query: sqlite3 positions.db "SELECT * FROM circuit_breaker_events ORDER BY id DESC LIMIT 5;"
  First5:  1. DO NOT override — this is intended safety behavior
           2. /health → confirm circuitBreaker: OPEN
           3. /positions history → check last 3–5 exits for SL cluster pattern
  Resume:  Wait slCircuitBreakerPauseMin (default 60min) — persists through restarts
           Telegram "✅ Hunter Circuit Breaker Reset" = safe to proceed
  Verify:  /health → circuitBreaker: CLOSED
           runtime-state.json: hunter-circuit-breaker key absent
  Escalate: Trips >3×/day → increase slCircuitBreakerCount (3→4) in user-config.json
  Manual Reset (emergency only): Delete hunter-circuit-breaker from runtime-state.json → restart

QC-2: DATA STALE / MARKET SNAPSHOT UNRELIABLE
  Detect:  Logs historySuccess: false | /providers → DexScreener DEGRADED
           OHLCV age > maxOhlcvStaleMinutes (15m: 90min, 1h: 180min)
  Act:     No action needed — entries auto-blocked, exits use on-chain price
  Resume:  Logs historySuccess: true → entry screening resumes automatically
  Verify:  /providers → all HEALTHY
  Escalate: DexScreener down >30min + Meteora price unreliable → /pause

QC-3: SWAP UNRESOLVED / manual_review
  Detect:  status='manual_review' in positions DB | "LIQUIDITY_TRAP" in logs
  Act:     1. /pause    2. Check on-chain via Meteora UI
           3. Manual close if open    4. Update DB status after
  Resume:  /resume after manual_review queue = 0
  Verify:  sqlite3 "SELECT COUNT(*) FROM positions WHERE status='manual_review';" → 0
  Escalate: Manual close fails on UI → contact Meteora support

QC-4: MAX HOLD EXITS CLUSTERING
  Detect:  Multiple "Max Hold Exit — CLOSE" Telegram alerts in short window
           exitTrigger='MAX_HOLD_EXIT' for many positions same hour
  First5:  1. No intervention — dead capital cleanup working correctly
           2. /status → confirm positions closed (not manual_review)
           3. Check: was regime SIDEWAYS_CHOP during holding period?
  Resume:  Bot auto-resumes entry after positions close
  Adjust:  If recurring weekly → reduce maxHoldHours: 6→4 in user-config.json
  Escalate: If clustering + circuit breaker fire together → /pause + root-cause review

QC-5: DAILY DRAWDOWN LIMIT BREACHED
  Detect:  Log "Daily loss limit reached" | dailyLossUsd >= dailyLossLimitUsd ($25 default)
           Telegram: "⛔ DAILY LOSS LIMIT REACHED — autonomous ops paused"
  First5:  1. Bot auto-pauses new entries (exits still fire on open positions)
           2. /status → check dailyLossUsd, open positions, open PnL
           3. /positions history → identify SL pattern or large loss causing breach
  Resume:  Next UTC midnight (auto-reset) OR /resume (manual — confirm root cause first)
  Verify:  /health → dailyLoss: LIMIT_REACHED | no new entries in logs after pause
  Escalate: Hits limit 3 consecutive days → reduce deployAmountSol or stopLossPct

REGIME BEHAVIOR QUICK REFERENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  BULL_TREND      → Entries allowed (Evil Panda / Deep Fishing)
  SIDEWAYS_CHOP   → Entries blocked by low ATR (await volatility)
  BEAR_DEFENSE    → ALL entries hard-blocked (policy: REGIME_BEAR_DEFENSE)
  LOW_LIQ_HIGH_RISK → Pool skipped (insufficient TVL or volume flow)

ENTRY BLOCK POLICY CODES:
  REGIME_BEAR_DEFENSE       classifyMarketRegime() === 'BEAR_DEFENSE'
  CIRCUIT_BREAKER_ACTIVE    hunter-circuit-breaker.pausedUntil > now (runtime-state.json)
  FAIL_SAFE_UNRELIABLE_DATA oracle/TA data unreliable (failSafeModeOnDataUnreliable=true)
  HTF_NULL_STRICT_ATR       1h data missing + atrPct < 2.0
  TREND_BEARISH             supertrend.trend === 'BEARISH'
  ATR_LOW                   atrPct < minAtrPctForEntry
  FEE_VELOCITY_DOWN         3-sample feeTvlRatio strictly descending

  SL_COOLDOWN_ACTIVE = Healer HOLD delay only — does NOT block Hunter entry.
  CIRCUIT_BREAKER_ACTIVE = Hunter entry block from persisted state.

  Exits always fire regardless of regime.
  Circuit breaker persists across restarts (runtime-state.json).
  Strategy confidence auto-adjusts via performanceHistory on each close.
