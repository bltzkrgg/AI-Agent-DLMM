## Final decisions

- New WATCH promotions must pause when deploy slots are saturated.
- Queue hold notifications for slot saturation must stay silent.
- OOR Telegram display should stay compact and cadence-driven, not verbose.
- Supertrend 15m bearish must remain a hard stop for entry unless the user explicitly changes that policy.
- Pool impact exit logic must remain a risk guard, not a generic panic rewrite.
- TA profit exits must respect `takeProfitMinNetPnlPct`; TA defensive exits can still close for risk control.
- Defensive TA exit scenario C must stay blocked while the position is still in range, and only becomes eligible on confirmed out-of-range-low conditions; in-range or out-of-range-high positions should keep waiting for bounce/TP or other normal exits.
- DLMM deploy range must stay active-bin-based per Meteora, but anchor provenance and range-adjust reasons should be observable in logs/state.
- Deploy range source should stay active-bin-relative (`min/max bin offset from active bin`) rather than deep percent-offset math; deploy range is now driven only by the two offsets and should not depend on a separate width-cap config.
- Deploy balance checks must fail before any chain-touching deploy step when effective SOL budget is insufficient, including position setup cost and overlapping in-flight deploy reservations.
- Deploy preflight should not use a hardcoded fee buffer; keep checks aligned to actual Meteora setup/rent costs instead.
- Final deploy proximity should stay simple: compare the frozen entry anchor against the latest live price/bin, allow only a small upside tolerance on fast bullish moves, and keep freshness strict instead of adding calm-candle grace.
- Final deploy must separate `snapshot can price/bin be read?` from `snapshot has full candle history?`. Missing full Meteora history may still block final ST/candle confirmation, but it should not kill the execution path before live price/bin proximity gets a chance to run.
- `liveSpotPrice` (open candle's close price) must remain in oracle OHLCV output as a separate field from `currentPrice` (last closed candle). `pendingDeployQueue.readLiveActiveBinState` must always prefer `liveSpotPrice` over `currentPrice` when available. Do not merge or collapse these two fields. `currentPrice` stays as fallback only. Supertrend and TA calculations must continue to use closed candles only and must not read `liveSpotPrice`.
- `/unblock` command only bypasses `BLACKLIST_LOCAL` gate. It must never bypass `STAGE_2_GMGN`, `MERIDIAN_VETO`, `FLAT_CONFIG_GATE`, or any other risk gate. The allowlist lives in `unblock.json` and is checked inside `isBlacklisted()` only. Do not add allowlist checks anywhere else in the pipeline. `/block` must always auto-remove the token from the allowlist if present, to prevent silent conflicts.
